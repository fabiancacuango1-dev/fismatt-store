                    🎓 CONCEPTOS TÉCNICOS (PARA ENTENDER)
════════════════════════════════════════════════════════════════════════════════

QUÉ ES GITHUB ACTIONS:
Servidor que GitHub mantiene gratis. Se ejecuta cuando haces push.
En tu caso: descarga código → copia a GitHub Pages → publica

QUÉ ES GITHUB PAGES:
Hosting estático GRATIS de GitHub. Sirve HTML, CSS, JS.
Tu URL: https://fabiancacuango1-dev.github.io/fismatt-store/

QUÉ ES EL ARCHIVO .github/workflows/deploy.yml:
Es un "script" que Le dice a GitHub Actions qué hacer automáticamente.

QUÉ SIGNIFICA "path: '.'":
Significa "sube TODO el contenido de la carpeta raíz (.)".
Incluye: index.html, css/, js/, assets/, etc.
Automáticamente ignora: .git, .github, node_modules, etc.

QUÉ SIGNIFICA "push branches [ main ]":
"Ejecuta este workflow cada vez que alguien hace push a rama main".

QUÉ SIGNIFICA "permissions: contents: read, pages: write":
"GitHub Actions puede leer tu código y escribir en Pages".

                    🚨 POSIBLES PROBLEMAS Y SOLUCIONES RÁPIDAS
════════════════════════════════════════════════════════════════════════════════

PROBLEMA 1: "Actions no corre / No aparece en pestaña Actions"

SOLUCIÓN:
→ Abre Settings → Actions → General
→ Selecciona "Allow all actions and reusable workflows"
→ Haz otro git push


PROBLEMA 2: "Workflow corre pero falla: 'index.html not found'"

SOLUCIÓN:
→ index.html DEBE estar en RAÍZ del repo
→ No debe estar en: /tienda-online/index.html
→ Debe estar en: /index.html
→ Si está mal, muévelo


PROBLEMA 3: "Desploy exitoso pero CSS/JS no cargan (página sin estilos)"

SOLUCIÓN:
→ Abre DevTools (F12)
→ Ve a Console
→ Mira errores GET css/ o GET js/
→ Si está en raíz, debería funcionar
→ Si no funciona, intenta con rutas absolutas:
   <link rel="stylesheet" href="/fismatt-store/css/styles.css">


PROBLEMA 4: "Dominio personalizado no funciona"

SOLUCIÓN:
→ Espera 24-48 horas (DNS lento)
→ Verifica CNAME en tu registrador: fabiancacuango1-dev.github.io
→ Habilita HTTPS en GitHub Pages (Settings → Pages)


PROBLEMA 5: "Quiero que NO se despliegue algunos archivos"

SOLUCIÓN:
→ Agrega .gitignore con:
   DEPLOY.md
   GITHUB_ACTIONS_*.txt
   etc.
→ Esos archivos no se suben
→ Así el Deploy es más limpio

TIP 1: Crea un .gitignore para no subir archivos innecesarios

$ cat > .gitignore << 'EOF'
.DS_Store
.env
.env.local
node_modules/
dist/
build/
*.log
.vscode/
.idea/
DEPLOY.md
GITHUB_ACTIONS_*
CORRECCIONES_PRE_DEPLOY.md
ANALISIS_TECNICO_COMPLETO.md
REFERENCIA_RAPIDA_DEPLOY.txt
EOF

Esto hace que esos archivos NO se suban a GitHub.


TIP 2: Checkea tu deploy desde Actions sin esperar

Si quieres forzar un deploy sin hacer cambios:
→ Ve a Actions
→ Selecciona "Deploy a GitHub Pages"
→ "Run workflow"
→ Selecciona "main"
→ "Run workflow"


TIP 3: Monitorea el estado en tiempo real

En Actions puedes ver:
- Cada paso que se está ejecutando
- Cuánto tiempo tardó cada paso
- Si algo falló, dónde exactamente
- Logs completos


TIP 4: Usa commits descriptivos

$ git commit -m "Update homepage text and colors"

Aparece en el historial de Actions. Ayuda a trackear cambios.
